# Vision

go-database ist ein „Hafen“ für Datenbanken — eine zentrale Schnittstelle zwischen Anwendungen und verschiedenen Datenbanksystemen. Es vereint sechs verschiedene DB-Engines unter einer einzigen REST-API, bietet ein vollwertiges Admin-Dashboard, ein rollenbasiertes Berechtigungssystem und einen Datentransfer zwischen unterschiedlichen Datenbank-Typen — alles in einer einzigen, portablen Binary.

# Datenbank-Plugins

Jeder DB-Typ wird durch ein eigenes Plugin implementiert, das das DBPlugin-Interface erfüllt — über die Plugin-Registry automatisch registriert.

★
PostgreSQL
pgx/v5 Pool, information_schema
★
MySQL
go-sql-driver, information_schema
★
MariaDB
go-sql-driver, kompatibel mit MySQL
★
SQLite
modernc.org (CGO-frei), PRAGMA
★
MongoDB
BSON-Aggregation, Schema-Inference
★
Redis
go-redis, KEYS, TYPE/TTL

Alle Plugins implementieren: Connect(), Ping(), Close(), Query(), Execute(), Tables(), Schema(), Databases(), CreateDatabase(), DropDatabase().

# Kern-Features

⚙

Einheitliche REST-API

53 Endpunkte mit einheitlichem JSON-Response-Format — { success, data, error, meta }.

⚙

6 DB-Engines

PostgreSQL, MySQL, MariaDB, SQLite, MongoDB, Redis — erweiterbar via Plugin-Interface.

⚙

RBAC Permission-System

19 Permission-Typen, 3 Default-Rollen (admin/developer/readonly), User-Overrides.

⚙

JWT + API-Keys

Session (JWT mit AES-256-GCM) und API-Keys (crypto/rand + SHA256 + bcrypt). Login-Rate-Limit.

⚙

SQL-Autocomplete

Trie-basiert mit Levenshtein-Fuzzy-Matching, Schema-Aware und Natural-Language-Intents (DE/EN).

⚙

Risikobewertung

SQL-Risk-Classifier (LOW/MEDIUM/HIGH), Bestätigung für riskante Queries.

⚙

Datentransfer

Cross-DB-Transfer mit auto Schema-Mapping, Dry-Run, Batch-Streaming.

⚙

Adaptives Dashboard

React 19 SPA mit responsivem Design, Dark/Light-Mode, DE/EN-i18n, konfigurierbarem Theme.

⚙

Single Binary

Frontend via embed.FS im Go-Binary. Keine separaten Dateien zum Deployment nötig.

⚙

Audit-Logging

Strukturiertes JSON-Logging aller Aktionen mit slog + X-Request-ID-Tracing.

# Authentifizierung

JWT (Session)

Erhalten via POST /auth/login (bcrypt). Token mit AES-256-GCM verschlüsselt. Konfigurierbare Laufzeit (Default: 60min).

API-Keys

32 Zufallsbytes (crypto/rand), SHA256-gehasst gespeichert. Nur bei Erstellung sichtbar. Perfekt für CI/CD.

Rate-Limiting

5 Fehlversuche am Login blockieren die IP exponentiell (1s, 2s, 3s...). Reset bei erfolgreichem Login.

# Quickstart

Docker (empfohlen)

git clone https://github.com/LangLi7/go-database
cd go-database
docker compose --profile samples up -d
# UI: http://localhost:8080
# Login: admin / admin

Lokale Entwicklung

# Backend
go build -o bin/go-database ./cmd/server/
./bin/go-database

# Frontend (separater Dev-Server)
cd web
npm install
npm run dev        # http://localhost:5173

# Tests
go test ./internal/... -count=1 -v

# Frontend Build + einbetten
make build         # baut UI + Server + kopiert dist

Konfiguration

# config/config.yaml
server:
  host: "127.0.0.1"
  port: 8080
auth:
  jwt_secret: "change-me-in-production"
  token_duration: 60

# Oder via Umgebungsvariablen:
export GODB_AUTH_JWT_SECRET=mein-geheimer-schluessel
export GODB_SERVER_PORT=9090
Standard-Login: admin / admin. Vor Produktion unbedingt das Passwort ändern und jwt_secret setzen!
