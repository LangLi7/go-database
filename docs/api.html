API Referenz

Basis-URL: http://localhost:8080/api/v1 — Einheitliches Response-Format: { success, data, error, meta }

go-database bietet 53 REST-Endpunkte für die Verwaltung von sechs Datenbank-Engines. Die API unterstützt zwei Authentifizierungsmethoden:

MethodeHeaderVerwendung
JWT (Session)Authorization: Bearer <token>Erhalten via /auth/login
API KeyX-API-Key: <key>Erstellt via /apikeys
Der Login-Endpunkt unterliegt einem Rate-Limit: nach 5 Fehlversuchen wird die IP blockiert.

Auth

POST /auth/login
Authentifiziert einen Benutzer und gibt ein JWT zurück. Rate-Limited.
Auth: none
// Request
{ "username": "admin", "password": "admin" }

// Response 200
{
  "success": true,
  "data": {
    "token": "eyJhbGciOiJIUzI1NiIs...",
    "user_id": "admin-001",
    "username": "admin",
    "role": "admin"
  }
}
POST /auth/refresh
Erneuert ein bestehendes JWT.
Auth: JWT
{ "token": "eyJhbGciOiJIUzI1NiIs..." }
GET /auth/verify
Verifiziert ein Token und gibt Benutzerinformationen zurück.
Auth: JWT
POST /auth/change-password
Ändert das Passwort des authentifizierten Benutzers.
Auth: JWT
{ "old_password": "admin", "new_password": "neuesPasswort" }
// Response: 204 No Content

Connections

GET /connections
Listet alle registrierten Datenbank-Verbindungen.
connections:list
[{
  "id": "a1b2c3d4", "name": "Meine PG-DB",
  "type": "postgres", "source": "external",
  "state": "connected", "latency_ms": 2,
  "tags": ["production"]
}]
POST /connections
Registriert eine neue Datenbank-Verbindung.
connections:create
// PostgreSQL / MySQL / MariaDB
{
  "name": "Meine DB", "type": "postgres",
  "host": "localhost", "port": 5432,
  "database": "sampledb", "user": "dev",
  "password": "dev123", "ssl": false,
  "tags": ["dev"]
}
// SQLite: { "type": "sqlite", "filepath": "./data/sample.db" }
// Response: 201 Created
GET /connections/:id
Details einer einzelnen Verbindung.
connections:list
DELETE /connections/:id
Entfernt eine Verbindung.
connections:delete
Response: 204 No Content
GET /connections/:id/ping
Prüft die Verbindung und misst die Latenz.
connections:list
{ "latency_ms": 2, "status": "ok" }

Query

POST /connections/:id/query
SELECT-Query. Automatische SQL-Typerkennung; nur SELECT/EXPLAIN/DESC etc. erlaubt.
connections:query
{ "query": "SELECT * FROM users WHERE id = 1" }

{
  "columns": ["id", "name", "email"],
  "rows": [[1, "Alice", "alice@..."]],
  "rows_affected": 1,
  "duration_ms": 3
}
POST /connections/:id/execute
INSERT/UPDATE/DELETE/DDL-Query.
connections:execute
{ "query": "UPDATE users SET name = 'Bob' WHERE id = 1" }
POST /execute/safe
Query mit Risikobewertung und zusätzlicher Sicherheitsprüfung.
connections:query
{ "connection_id": "...", "sql": "DROP TABLE users", "confirm_high": true }

WebSocket

WS /ws/query/:id
Streaming-Query-Endpunkt über WebSocket. Bidirektionale Kommunikation für Echtzeit-Datenbankabfragen.
connections:query
// Client sendet:
{ "type": "query", "query": "SELECT * FROM users", "req_id": "abc123" }

// Server antwortet asynchron:
{ "type": "result", "req_id": "abc123", "success": true,
  "data": { "columns": ["id","name"], "rows": [[1,"Alice"]], "rows_affected": 1, "duration_ms": 3 } }

// Fehler:
{ "type": "result", "req_id": "abc123", "success": false, "error": "query failed: ..." }

// Execute (INSERT/UPDATE/DELETE):
{ "type": "execute", "query": "UPDATE users SET name='Bob' WHERE id=1", "req_id": "xyz" }

// Ping:
{ "type": "ping", "req_id": "..." } → { "type": "pong", "req_id": "..." }

Python Beispiel

Vollständiges Skript in docs/examples/python/api_test.py:

import requests

BASE = "http://localhost:8080/api/v1"

# 1. Login — JWT holen
r = requests.post(f"{BASE}/auth/login", json={
    "username": "admin", "password": "admin"
})
token = r.json()["data"]["token"]
headers = {"Authorization": f"Bearer {token}"}

# 2. SQLite-In-Memory-DB anlegen
r = requests.post(f"{BASE}/connections", headers=headers, json={
    "name": "Test", "type": "sqlite", "filepath": ":memory:"
})
conn_id = r.json()["data"]["id"]

# 3. Query ausführen
r = requests.post(f"{BASE}/connections/{conn_id}/execute", headers=headers, json={
    "query": "CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)"
})
r = requests.post(f"{BASE}/connections/{conn_id}/query", headers=headers, json={
    "query": "SELECT * FROM users"
})
print(r.json()["data"]["rows"])  # → []
