go-database Dashboard

Die Dashboard-Seite ist der Startbildschirm der Web-Oberfläche. Sie liefert auf einen Blick alle wichtigen Kennzahlen und den Status der Datenbank-Verbindungen.

◑

Dashboard-Ansicht (Dark Theme)

web/src/pages/Dashboard.tsx

Das Dashboard wird unter der Route / angezeigt und ist die erste Seite, die nach dem Login erscheint. Es kombiniert Statistik-Karten mit einer detaillierten Verbindungstabelle.

Datenfluss

Das Dashboard bezieht Daten aus zwei parallelen API-Aufrufen:

Mount → Promise.all → listConnections() → setConnections
Mount → Promise.all → listUsers() → setUsers

Beide Requests laufen parallel über Promise.all und setzen nach erfolgreichem Abschluss den loading-State auf false.

// Vereinfachter Datenfluss
useEffect(() => {
  let cancelled = false
  setLoading(true)

  Promise.all([
    api.listConnections(),
    api.listUsers()
  ]).then(([connRes, userRes]) => {
    if (cancelled) return
    if (connRes.success) setConnections(connRes.data || [])
    if (userRes.success) setUsers(userRes.data || [])
    setLoading(false)
  })

  return () => { cancelled = true }
}, [])

Loading-Zustand

Während die Daten geladen werden, zeigt ein zentrierter Spin-Loader (Ant Design) an:

Routing

Das Dashboard ist in der App.tsx als Standard-Route registriert:

// web/src/App.tsx (Auszug)
<Routes>
  <Route path="/" element={<Dashboard />} />
  <Route path="/connections" element={<Connections />} />
  <Route path="/explorer" element={<Explorer />} />
  <Route path="/query" element={<QueryEditor />} />
  <Route path="/admin/users" element={<AdminUsers />} />
  <Route path="/admin/roles" element={<AdminRoles />} />
  <Route path="/admin/apikeys" element={<APIKeys />} />
  <Route path="/admin/settings" element={<Settings />} />
  <Route path="*" element={<Navigate to="/" replace />} />
</Routes>
Alle nicht gefundenen Routen leiten auf das Dashboard (/) weiter. Die gesamte App setzt eine gültige Authentifizierung voraus — bei fehlendem Token erfolgt eine Weiterleitung zur Login-Seite.

Embedding & Deployment

Das gebaute Frontend-Dashboard wird zur Build-Zeit in den Go-Binary eingebettet:

web/src/ → vite build → web/dist/ → kopieren nach → internal/dashboard/dist/ 
// internal/dashboard/embed.go
package dashboard

import (
    "embed"
    "io/fs"
    "net/http"
)

//go:embed dist/*
var distFS embed.FS

func FS() (http.FileSystem, error) {
    return http.FS(sub), nil
}

Im Go-Server werden die Dashboard-Dateien wie folgt ausgeliefert:

// main.go (Auszug)
dashFS, _ := dashboard.FS()
r.StaticFS("/assets", dashFS)
r.StaticFile("/favicon.svg", ...)
r.StaticFile("/favicon.ico", ...)
r.NoRoute(func(c *gin.Context) {
    if strings.HasPrefix(c.Request.URL.Path, "/api/") {
        c.JSON(404, gin.H{"error": "not found"})
        return
    }
    c.FileFromFS("index.html", dashFS)
})
Dadurch ist das gesamte Frontend im finalen Go-Binary enthalten. Es wird keine separate Webserver-Konfiguration oder Datei-Pfad benötigt — eine einzelne Binary reicht aus.
